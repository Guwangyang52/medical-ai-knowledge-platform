# 第一个分析执行任务

本文演示如何用 starter 中的模板完成一次最小执行闭环。

建议先跑 Python demo，因为它不依赖 R 包、JAGS 或系统级统计环境；然后再跑连续数据 Meta 分析 R demo。

## A. Python demo

### 1. 检索模板

在 `execution-platform` 中运行：

```powershell
python .\platform-core\scripts\search_kb.py "Python 表格数据摘要" --json
```

应命中：

```text
wiki/analyses/Python表格数据摘要/
```

### 2. 创建任务目录

```powershell
python .\platform-core\scripts\create_analysis_project.py "python demo" --template-dir "..\my-vault\wiki\analyses\Python表格数据摘要" --copy-template-code --task-id python-demo
```

任务会创建在：

```text
execution-platform/projects/python-demo/
```

### 3. 准备数据

```powershell
Copy-Item "..\my-vault\wiki\analyses\Python表格数据摘要\data\sample_table.csv" ".\projects\python-demo\data\sample_table.csv"
```

### 4. 运行 Python 脚本

```powershell
python .\platform-core\scripts\run_python_task.py ".\projects\python-demo" --script "src/analysis.py"
```

Python runner 会自动生成：

- `logs/stdout.log`
- `logs/stderr.log`
- `output/summary.json`
- `report/output_manifest.json`
- `report/result_summary.md`
- `run_manifest.yaml`

## B. R demo：连续数据 Meta 分析

### 1. 检索模板

```powershell
python .\platform-core\scripts\search_kb.py "连续变量 Meta 分析" --json
```

应命中：

```text
wiki/analyses/连续数据的Meta分析/
```

### 2. 创建任务目录

```powershell
python .\platform-core\scripts\create_analysis_project.py "continuous meta demo" --template-dir "..\my-vault\wiki\analyses\连续数据的Meta分析" --copy-template-code --task-id continuous-meta-demo
```

### 3. 准备数据

```powershell
Copy-Item "..\my-vault\wiki\analyses\连续数据的Meta分析\data\sample_continuous_meta.csv" ".\projects\continuous-meta-demo\data\sample_continuous_meta.csv"
```

### 4. 运行 R 脚本

```powershell
python .\platform-core\scripts\run_r_task.py ".\projects\continuous-meta-demo" --script "src/analysis.R"
```

### 5. 归档结果

```powershell
python .\platform-core\scripts\archive_result.py ".\projects\continuous-meta-demo"
```

归档后查看：

```text
projects/continuous-meta-demo/report/result_summary.md
projects/continuous-meta-demo/report/knowledge_candidate.md
```

## C. 是否沉淀

如果任务只是临时分析，不需要写回 wiki。

如果任务产生了可复用流程，应先生成 `knowledge_candidate.md`，再由用户决定是否沉淀到：

```text
my-vault/wiki/
my-skills/
```

不要把任务输出直接复制进 `wiki/`。

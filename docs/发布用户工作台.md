# 发布用户工作台

本文面向项目维护者，说明如何从开发者仓库生成普通用户可直接解压使用的工作台包。

## 分发模型

本仓库只维护一套源码，但提供两种入口：

- 开发者入口：clone 本仓库，修改 `platform-core/`、`starter-vault/`、`starter-skills/`、文档和 bootstrap 脚本。
- 普通用户入口：下载 GitHub Releases 中的 `MedicalAIKnowledgeWorkspace-v<version>.zip`，解压后双击 `workspace.code-workspace`。

Release zip 内部结构为：

```text
MedicalAIKnowledgeWorkspace/
├─ my-vault/
├─ execution-platform/
├─ my-skills/
├─ workspace.code-workspace
├─ start.ps1
└─ README_先打开我.md
```

## 本地生成

在仓库根目录运行：

```powershell
.\tools\build_user_workspace.ps1 -Version 0.1.0
```

生成结果：

```text
dist/MedicalAIKnowledgeWorkspace/
dist/MedicalAIKnowledgeWorkspace-v0.1.0.zip
```

## 自动发布

推送 tag 后，GitHub Actions 会自动构建 zip 并附加到对应 Release：

```powershell
git tag v0.1.0
git push origin v0.1.0
```

也可以在 GitHub Actions 页面手动运行 `Release user workspace` workflow。

## 发布前检查

建议发布前至少检查：

```powershell
.\tools\build_user_workspace.ps1 -Version 0.1.0
cd .\dist\MedicalAIKnowledgeWorkspace\execution-platform
python .\platform-core\scripts\search_kb.py "连续变量 Meta 分析" --json
python .\platform-core\scripts\validate_kb.py --json
```

确认 zip 不包含真实患者数据、私有课题数据、未授权 PDF/Word/PPT、API key、本地任务输出或日志。
